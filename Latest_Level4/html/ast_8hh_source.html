7  About:
8 
9  Originally implemented by Tanu Kanvar and Uday Khedker (http://www.cse.iitb.ac.in/~uday)
10  for the courses cs302+cs316 Language Processors (theory and lab) at IIT Bombay.
11 
12  Initial release date Jan 15, 2013.
13 
14  Updated by N. Venkatesh in Jan 2019. The main updates included
15  - Changing the input language from a basic-block-based GCC GIMPLE dump to a subset of C.
16  - Replacing the flexc++ and bisonc++ scripts by flex and bison.
17  - Supporting print operation for variables.
18 
19  Updated by Uday Khedker with the help of Nisha Biju and Saari Rajan in Jan 2020. The main
20  updates include
21  - Introduction of three address code (TAC) phase between abstract syntax tree (AST)
22  and Register Transfer Language (RTL) which was originally called icode.
23  - Introduction of string data type with constant strings.
24  - Support for read operation for integer and floating point variables, and print
25  operation for all variables (including string variables) and string constants.
26  - Significant changes in command line switches for printing each representation (tokens,
27  AST, TAC, RTL, and assembly) as well as stopping after each phase (scanning, parsing,
28  AST construction (i.e. semantic analysis), TAC generation and RTL generation).
29  - Substantial clean up of the code including removal of AST to icode translation,
30  and interpretation (which was buggy and had not been tested thorougly after the front
31  end and the input langauge changed in 2019).
32 
33  Copyrights reserved by Uday Khedker. This implemenation has been made available purely
34  for academic purposes without any warranty of any kind.
